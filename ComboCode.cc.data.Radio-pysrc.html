 36          created from scratch at given location. 
 37           
 38          The name of the database is radio_data.db by default. Can be changed if  
 39          needed, but ComboCode does not work with this other name. 
 40           
 41          If nothing more is requested, a new database is essentially empty.  
 42           
 43          The option to parse the content of the target folder is available: This  
 44          will add entries to the database for the stars in the folder and the  
 45          'simple' molecules and transitions.  
 46                   
 47          Once a db is created, you can perform several methods on the database  
 48          specific to radio data management. 
 49           
 50          Examples: 
 51          For data located in /home/robinl/Data/Molecular/ 
 52          >>> from cc.data import Radio 
 53          >>> db = Radio.Radio('/home/robinl/Data/Molecular/',auto_parse=1) 
 54          >>> db.sync() 
 55   
 56          This will create a database, automatically search your data folder for  
 57          data files that the script can recognize, and save them in the  
 58          database. The final command ( .sync() ) will then save the database to 
 59          your hard disk in the same folder, under 'radio_data.db'. Later when  
 60          you run Radio.Radio('/home/robinl/Data/Molecular/') again, it will load 
 61          that same database that still contains all those datafiles and  
 62          TRANSITION references. 
 63   
 64          The database is structured per star_name and per transition definition. 
 65          So for instance, you will find the file whya_co32_APEX.fits in a list  
 66          under 
 67          db['whya']['TRANSITION=12C16O 0 3 0 0 0 2 0 0 APEX 0.0'] 
 68          where you will find 
 69          ['whya_co32_APEX.fits'] 
 70   
 71          If multiple data files are available for the same star, the same  
 72          transition and the same telescope, the multiple data files will also be 
 73          contained in this list. Note that the transition definition is quite  
 74          specific: 1 space between entries, and 11 entries total. It is  
 75          identical to the input in your combocode inputfile (excluding the final 
 76          number n_quad, which is model input and has nothing to do with your  
 77          data). 
 78   
 79          Lastly, you can do multiple things with this database. 
 80          >>> db.addData(star_name='whya',trans='TRANSITION=12C16O 0 3 0 0 0 2 0 0 JCMT 0.0',\ 
 81                         filename='whya_co32_Maercker_new_JCMT.fits') 
 82          will add a new entry to the database if it is not there. This is useful 
 83          for those molecules/filenames which have not been automatically found  
 84          by the auto_parse (for instance SO2 data files do not stick to the file

 86          with this method you can still add them whichever way you want. 
 87   
 88          >>> db.removeData(star_name='whya',trans='TRANSITION=12C16O 0 3 0 0 0 2 0 0 JCMT 0.0') 
 89          removes a whole transition from the database. Alternatively 
 90   
 91          >>> db.removeData(star_name='whya',filename='whya_co32_Maercker_new_JCMT.fits') 
 92          removes a single filename from the database. (does not delete the file) 
 93

 95          in the database (so you know which you have to add manually): 
 96          >>> db.checkFolder()
